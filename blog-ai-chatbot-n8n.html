Building an AI-Powered Chatbot with n8n and RAG

A step-by-step guide to creating a document-aware chatbot using Retrieval Augmented Generation

AI Engineer

Published on May 25, 2025

In this comprehensive guide, I'll walk you through the process of building an AI-powered chatbot that can answer questions based on your documents, using n8n for workflow automation and Retrieval Augmented Generation (RAG) for enhanced responses.

Introduction to RAG-Powered Chatbots

Retrieval Augmented Generation (RAG) is a technique that enhances large language models by providing them with relevant context from a knowledge base. This approach significantly improves the accuracy and relevance of AI responses, especially when dealing with domain-specific questions or proprietary information.

By combining the power of n8n's workflow automation with RAG, we can create a chatbot that:

  • Processes and indexes documents from various sources
  • Retrieves relevant information based on user queries
  • Generates accurate, contextually appropriate responses
  • Integrates seamlessly with existing systems

System Architecture

Our chatbot system consists of several key components:

  1. Document Processor: Handles the ingestion and parsing of various document formats
  2. Vector Database: Stores document embeddings for semantic search
  3. n8n Workflow: Orchestrates the entire process from document ingestion to response generation
  4. LLM Integration: Connects to language models for generating responses
  5. User Interface: Provides a chat interface for user interaction

Setting Up the Environment

Let's start by setting up our development environment. We'll need:

  • n8n installed and running
  • A vector database (we'll use Qdrant for this tutorial)
  • Access to an LLM API (OpenAI or a self-hosted alternative)
  • Basic knowledge of JavaScript and Python

Installing n8n

First, let's install n8n using npm: 

npm install n8n -g
n8n start

Setting Up Qdrant

We'll use Docker to run Qdrant: 

docker pull qdrant/qdrant
docker run -p 6333:6333 -p 6334:6334 \
    -v $(pwd)/qdrant_storage:/qdrant/storage \
    qdrant/qdrant

Document Processing Workflow

Now, let's create our n8n workflow for processing documents:

Step 1: Document Ingestion

We'll start by creating a workflow that watches a specific folder for new documents: 

// n8n workflow - Document Ingestion
[
  {
    "name": "Watch Folder",
    "type": "n8n-nodes-base.filesWatcher",
    "parameters": {
      "path": "/path/to/documents",
      "fileExtensions": [
        "pdf",
        "docx",
        "txt"
      ]
    }
  },
  {
    "name": "Process Document",
    "type": "n8n-nodes-base.function",
    "parameters": {
      "functionCode": `
        // Document processing code
        const filePath = $input.item.path;
        const fileType = filePath.split('.').pop();
        
        // Process based on file type
        let text;
        if (fileType === 'pdf') {
          // PDF processing logic
          text = extractTextFromPDF(filePath);
        } else if (fileType === 'docx') {
          // DOCX processing logic
          text = extractTextFromDOCX(filePath);
        } else {
          // Plain text
          text = readTextFile(filePath);
        }
        
        return {
          json: {
            filePath,
            fileType,
            text,
            chunks: chunkText(text, 1000, 200)
          }
        };
      `
    }
  }
]

Step 2: Text Embedding

Next, we'll create embeddings for each text chunk: 

// n8n workflow - Text Embedding
{
  "name": "Create Embeddings",
  "type": "n8n-nodes-base.openAi",
  "parameters": {
    "authentication": "apiKey",
    "operation": "createEmbedding",
    "text": "={{ $json.chunks }}",
    "model": "text-embedding-ada-002"
  }
}

Step 3: Store in Vector Database

Now, let's store these embeddings in Qdrant: 

// n8n workflow - Store in Qdrant
{
  "name": "Store in Qdrant",
  "type": "n8n-nodes-base.httpRequest",
  "parameters": {
    "url": "http://localhost:6333/collections/documents/points",
    "method": "PUT",
    "authentication": "none",
    "jsonParameters": true,
    "bodyParametersJson": `{
      "points": [
        {
          "id": "={{ $json.id }}",
          "vector": "={{ $json.embedding }}",
          "payload": {
            "text": "={{ $json.chunk }}",
            "source": "={{ $json.filePath }}",
            "metadata": {
              "fileType": "={{ $json.fileType }}"
            }
          }
        }
      ]
    }`
  }
}

Building the Chatbot Interface

Now that we have our document processing pipeline set up, let's create the chatbot interface:

HTML Structure

<div class="chatbot-container">
  <div class="chatbot-messages" id="chatMessages"></div>
  <div class="chatbot-input">
    <input type="text" id="userInput" placeholder="Ask about the documents...">
    <button id="sendButton">Send</button>
  </div>
</div>

JavaScript for the Chatbot

// Chatbot functionality
document.addEventListener('DOMContentLoaded', () => {
  const chatMessages = document.getElementById('chatMessages');
  const userInput = document.getElementById('userInput');
  const sendButton = document.getElementById('sendButton');

  // Add event listeners
  sendButton.addEventListener('click', sendMessage);
  userInput.addEventListener('keypress', (e) => {
    if (e.key === 'Enter') sendMessage();
  });

  // Function to send message
  async function sendMessage() {
    const message = userInput.value.trim();
    if (!message) return;

    // Add user message to chat
    addMessageToChat('user', message);
    userInput.value = '';

    // Show typing indicator
    const typingIndicator = addTypingIndicator();

    try {
      // Call the n8n webhook to process the message
      const response = await fetch('http://localhost:5678/webhook/chatbot', {
        method: 'POST',
        headers: {
          'Content-Type': 'application/json'
        },
        body: JSON.stringify({ query: message })
      });

      const data = await response.json();
      
      // Remove typing indicator
      typingIndicator.remove();
      
      // Add bot response to chat
      addMessageToChat('bot', data.response, data.sources);
    } catch (error) {
      console.error('Error:', error);
      typingIndicator.remove();
      addMessageToChat('bot', 'Sorry, I encountered an error processing your request.');
    }
  }

  // Function to add message to chat
  function addMessageToChat(sender, message, sources = []) {
    const messageElement = document.createElement('div');
    messageElement.className = `message ${sender}-message`;
    
    // Message content
    messageElement.innerHTML = `
${message}
`;
    
    // Add sources if available
    if (sources && sources.length > 0) {
      const sourcesElement = document.createElement('div');
      sourcesElement.className = 'message-sources';
      sourcesElement.innerHTML = '
Sources:
    ' + 
        sources.map(source => `
  • ${source}
    • `).join('') + 
        '
';
      messageElement.appendChild(sourcesElement);
    }
    
    chatMessages.appendChild(messageElement);
    chatMessages.scrollTop = chatMessages.scrollHeight;
  }

  // Function to add typing indicator
  function addTypingIndicator() {
    const typingElement = document.createElement('div');
    typingElement.className = 'typing-indicator';
    typingElement.innerHTML = '';
    chatMessages.appendChild(typingElement);
    chatMessages.scrollTop = chatMessages.scrollHeight;
    return typingElement;
  }
});

Creating the RAG Query Workflow

Finally, let's create the n8n workflow that handles user queries: 

// n8n workflow - RAG Query Handler
[
  {
    "name": "Webhook",
    "type": "n8n-nodes-base.webhook",
    "parameters": {
      "path": "chatbot",
      "responseMode": "lastNode",
      "options": {
        "responseHeaders": {
          "entries": [
            {
              "name": "Content-Type",
              "value": "application/json"
            }
          ]
        }
      }
    }
  },
  {
    "name": "Create Query Embedding",
    "type": "n8n-nodes-base.openAi",
    "parameters": {
      "authentication": "apiKey",
      "operation": "createEmbedding",
      "text": "={{ $json.query }}",
      "model": "text-embedding-ada-002"
    }
  },
  {
    "name": "Search Vector DB",
    "type": "n8n-nodes-base.httpRequest",
    "parameters": {
      "url": "http://localhost:6333/collections/documents/points/search",
      "method": "POST",
      "authentication": "none",
      "jsonParameters": true,
      "bodyParametersJson": `{
        "vector": "={{ $json.embedding }}",
        "limit": 5,
        "with_payload": true
      }`
    }
  },
  {
    "name": "Prepare Context",
    "type": "n8n-nodes-base.function",
    "parameters": {
      "functionCode": `
        // Extract relevant text from search results
        const results = $input.json.result;
        const context = results.map(item => item.payload.text).join('\\n\\n');
        const sources = results.map(item => item.payload.source);
        
        return {
          json: {
            query: $input.json.query,
            context,
            sources
          }
        };
      `
    }
  },
  {
    "name": "Generate Response",
    "type": "n8n-nodes-base.openAi",
    "parameters": {
      "authentication": "apiKey",
      "operation": "createChatCompletion",
      "messages": [
        {
          "role": "system",
          "content": "You are a helpful assistant that answers questions based on the provided context. If the answer is not in the context, say that you don't know."
        },
        {
          "role": "user",
          "content": "=== CONTEXT ===\n{{ $json.context }}\n\n=== QUESTION ===\n{{ $json.query }}"
        }
      ],
      "model": "gpt-4",
      "options": {
        "temperature": 0.3
      }
    }
  },
  {
    "name": "Format Response",
    "type": "n8n-nodes-base.function",
    "parameters": {
      "functionCode": `
        return {
          json: {
            response: $input.json.choices[0].message.content,
            sources: $input.json.sources
          }
        };
      `
    }
  }
]

Conclusion

By following this guide, you've created a powerful AI chatbot that can answer questions based on your documents. The combination of n8n's workflow automation and RAG technology enables you to build sophisticated AI applications without extensive coding.

This approach can be extended to handle various document types, integrate with different LLMs, and connect to multiple data sources. The possibilities are endless!
